The app is being migrated from a static prototype (localStorage + hardcoded `lib/products.ts`/`lib/cart.ts`/`lib/account.ts`) to headless Shopify.

- Store: `iby1ui-j5.myshopify.com` (shop name "AUGUSTA NEWHAM"). Storefront API version `2025-01`.
- Custom app is a new **Dev Dashboard** app (client_id `57eb…`). Tokens are minted via `client_credentials` (form-encoded) at `/admin/oauth/access_token` → 24h `shpat_` admin token; the permanent Storefront token was created via Admin REST `storefront_access_tokens.json`. The secret (`shpss_…`) was exposed in chat and should be rotated.
- `lib/shopify.ts` = server-side Storefront GraphQL client (`shopifyFetch`) + `getProducts()` / `getProduct(handle)` returning a `ShopifyProduct` shape. Token read from `STOREFRONT_ACCESS_TOKEN` in `.env.local` (server-only, not NEXT_PUBLIC).
- Customer auth uses the **Storefront Customer API** (FINAL decision 2026-06-23). We briefly built the Customer Account API (OAuth/PKCE) but reverted: it can't be configured from a **custom app** — its OAuth `client_id` only comes from the Headless/Hydrogen sales channel, so the redirect looped forever. The store is a custom app (not headless), and the `unauthenticated_read_customers`/`unauthenticated_write_customers` scopes were already accepted → Storefront Customer API is the right fit. Email/password forms are built by us; `customerAccessToken` stored in an httpOnly cookie. All customer GraphQL goes through `shopifyFetch` (lib/shopify.ts), `revalidate:0`.
  - Flow → mutation/query: login=`customerAccessTokenCreate`; signup=`customerCreate` (then auto-login); logout=`customerAccessTokenDelete`; forgot=`customerRecover`; reset=`customerResetByUrl`; profile read+orders+addresses=`customer(customerAccessToken:){ firstName lastName email phone defaultAddress addresses orders }`; profile edit (incl. email/phone/password)=`customerUpdate(customerAccessToken,customer:CustomerUpdateInput!)`; addresses=`customerAddress{Create,Update,Delete}(customerAccessToken,…,MailingAddressInput!)` + `customerDefaultAddressUpdate`. Mutations return `customerUserErrors{field message code}`.
  - **Built (2026-06-23, typechecks + lints clean, untested w/o products published + real customer):** `lib/session.ts` (token cookie `an_customer_token`), `lib/customer.ts` (queries+mutations on shopifyFetch), `lib/account-actions.ts` (server actions incl. login/register/recover/reset/logout/profile/address), `middleware.ts` (gates `/account/*` → `/login`), pages `app/{login,register,forgot-password,reset-password}/page.tsx` + form components, `app/api/auth/logout/route.ts`, `app/account/page.tsx` (redirects to /login on missing/invalid token). `AccountDashboard` props-driven (live profile/addresses/orders, email+phone editable). DELETED the Customer Account API code (`lib/customer-auth.ts`, `lib/customer-api.ts`, `app/api/auth/login` + `/callback`) and the `lib/account.ts` prototype.
  - **Prereq:** add `unauthenticated_read_customers` + `unauthenticated_write_customers` to the custom app's Storefront scopes (user says accepted). Forgot-password reset emails link to the Shopify online-store reset page by default — to land on our `/reset-password`, the "Customer account password reset" email template must point at our domain (reset page calls `customerResetByUrl`).

- **TOKENLESS PIVOT (2026-06-23):** The custom-app `STOREFRONT_ACCESS_TOKEN` returns **0 products** — products are published to the Online Store channel but NOT to the custom app's sales channel, so the tokened query is filtered to an empty set (proven: tokenless query returns products, tokened returns `[]`; an empty list, not ACCESS_DENIED). Reference impl `C:\Users\USER\Documents\webdev\work\920storefront\lib\shopify.ts` runs Storefront **tokenless** (token only attached if present) — "tokenless works for products/collections/cart." So we made `shopifyFetch(query, vars, revalidate, withToken=true)`: `getProducts`/`getProduct` and all `lib/shopify-cart.ts` calls pass `withToken=false` (tokenless); `lib/customer.ts` keeps the token (auth needs it). **Metafields are DISABLED** — tokenless can't read them (`unauthenticated_read_metafields` requires the token, but the token returns no products). The `custom.*` metafield block is commented out in `PRODUCT_FRAGMENT`; brand fields (tagline/composition/care/details) now resolve empty, `description` still comes from Shopify. To restore metafields: publish products to the token's channel, re-add the metafield block, fetch those products `withToken=true`.

**Open blockers (as of 2026-06-22):**
1. Products return empty from the Storefront API — they're imported as **draft** and not **published to the headless sales channel**. Must publish before anything renders.
2. App scopes don't match the agreed 6: the created token is missing `read_collection_listings`, `read_product_metafields`, and the cart scopes (`read/write_carts`), and over-includes customer/content/metaobject scopes.

Agreed Storefront scopes for this app: `unauthenticated_read_product_listings`, `read_collection_listings`, `read_product_metafields`, `read_product_inventory`, `write_carts`, `read_carts`. Checkout uses Cart API + `cart.checkoutUrl` (the deprecated checkout scopes are not used). See [[seo-and-routes-architecture]] for the route/category structure the catalog maps onto.
